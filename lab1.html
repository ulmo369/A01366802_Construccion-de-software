  3. ¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?
    • GET: Solicita una representación del recurso especificado. Debe usarse para recuperar datos
    • HEAD: Provee información general (metadatos) acerca del documento (Titulo, script, etc)
    • POST: Envía una entidad a un recurso
    • PUT: Crea un nuevo elemento o reemplaza una representación del elemento de destino con los datos de la petición
    • PATCH: Aplica modificaciones a un recurso
    • DELETE: Borra un recurso
    Fuente: www.freecodecamp.org

  9. ¿Cuáles son las diferencias principales entre HTML 4 y HTML5?
    • HTML5 define una sintaxis que es compatible con HTML4, algo qeu HTML 4 no hace.
    • Se introducen nuevos elementos, como: section, article, aside, header, footer, etc.
    • Algunos atributos cambian, como: type, name, summary, etc.
    • Para definir el juego de caracteres se introduce un nuevo atributo para la etiqueta ”UTF-8″
    Fuente: blogs.ua.es

  10. ¿Qué componentes de estructura y estilo tiene una tabla?
    • table: Son datos con más de una dimensión.
    • caption: Dar un título a tu tabla
    • col: Columna de una tabla
    • colgroup: Grupo de columnas
    • thead: Delimitar el encabezado de la tabla
    • tfoot: Delimita la parte de la tabla correspondiente al pie de página
    • tbody: delimita las otras partes del contenido de la tabla que no están en el encabezado o en el pie de página de la tabla
    • td: Celda con datos de una tabla
    • tr: Fila de celdas en una tabla
    • th: Celda encabazado de una tabla
    Fuente: www.htmlquick.com

  11. ¿Cuáles son los principales controles de una forma HTML5?
    • form: Representa un formulario
    • fieldset: Controles
    • label: Titulo de un control de un form
    • legend: Titulo de un fieldset
    • input: Espacio donde el usuario es capas de editar y escribir
    • output: Resultado de cualquier calculo
    • select: Permite seleccionar una opción
    Fuente: developer.mozilla.org

  13. ¿Cuál es el ciclo de vida de los sistemas de información? ¿Cuál es el ciclo de desarrollo de sistemas de información?
    • El ciclo de vida de un sistema de información es un enfoque por fases del análisis y diseño que sostiene que los sistemas son desarrollados de mejor manera mediante el uso de un ciclo especifico de actividades del analista y del usuario.
    • El ciclo de vida de desarrollo de un sistema (SDLC) es un proceso lógico utilizado en el mundo del Desarrollo de Software sistemas para desarrollar un sistema de información, incluidos los requisitos, la validación, formación, como los usuarios (interesados) en la propiedad.
